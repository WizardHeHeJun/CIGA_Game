# /learn —— 经验沉淀

把这次会话里学到的、会复用的东西固化下来，让下次（任何会话）自动受益。

线索：$ARGUMENTS（为空则回顾本会话中用户的纠正、踩的坑、确认的做法）

## 分诊：按类型归位
| 学到的东西 | 写到哪 |
|------------|--------|
| 用户纠正了一个**会复发**的错误（lint 查不出：架构/生命周期/Unity 行为） | `ai-shared/pitfalls.md`（按标准格式：错误/修复/说明/规则） |
| 命名 / API 用法 / 序列化等**可正则检测**的错误 | 追加 `.claude/hooks/csharp_lint/rules.json` 一条规则（四层过滤），并跑测试确认不误报 |
| 需要长期遵守的**规范** | 对应 `.claude/rules/*.md` 增一条（保持精简，<150 行） |
| 用户**偏好** / 工作方式 | 写记忆（feedback 类型，含 Why） |
| 项目**动态 / 决策背景** | 写记忆（project 类型，相对日期转绝对） |

## 原则
- **lint 能查的不要重复写进 pitfalls/PRP**——否则注意力稀释（双重保障的前提是职责互补）。
- 一条经验只落一处主位置；相关处用引用链接。
- 加了 lint 规则后，考虑 `/sync-evals` 生成对应回归用例。

## 收尾
列出本次落了哪些条目、各在什么文件，方便用户 review。
